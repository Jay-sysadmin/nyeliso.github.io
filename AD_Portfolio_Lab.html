Step-by-Step Process
1
Open Server Manager Dashboard
Launched Server Manager on Windows Server 2019. Navigated to Add Roles and Features and selected the Active Directory Domain Services (AD DS) role to install.
2
Promote Server to Domain Controller
After AD DS installed, used the post-install wizard to promote the server to a Domain Controller. Created a new forest with the domain name jay.local. Set the DSRM (Directory Services Restore Mode) password and completed the promotion.
Domain: jay.local
3
Open Active Directory Users and Computers (ADUC)
Accessed ADUC via Tools → Active Directory Users and Computers in Server Manager. This is the main tool for managing all AD objects. Confirmed the domain structure was visible: jay.local with default containers.
Tools → Active Directory Users and Computers (dsa.msc)
4
Create Organisational Units (OUs)
Right-clicked jay.local → New → Organisational Unit. Created two OUs: Accounts (to hold user accounts) and Groups (to hold security groups). OUs are like folders that help organise and apply different policies to different sets of users.
5
Create User Accounts
Navigated to the Accounts OU, right-clicked → New → User. Created a new user following the naming convention (first name + last initial). Set a temporary password and required the user to change it at first login.
Example: JoE Martin → username: jmartin
6
Create Security Groups
Navigated to the Groups OU, right-clicked → New → Group. Created four Security Groups: Finance, HR, IT, and Marketing. Security groups control which resources (folders, printers, apps) each department can access.
7
Add User to Finance Group
Double-clicked the Finance group → Members tab → Add. Added JoE Martin (from jay.local/Accounts). This gives Martin access to Finance department resources. Verified the membership was saved correctly.
Finance Properties → Members → Add → JoE Martin
8
Join Client Computer to Domain & Test Login
On a client Windows machine, navigated to System Properties → Computer Name → Change. Changed from Workgroup to Domain, entered jay.local. Provided domain admin credentials. After reboot, signed in using the domain user account to confirm authentication was working.
sysdm.cpl → Computer Name → Change → Domain: jay.local

Key Concepts Practised
🏗️ Forest & Domain creation — A new AD forest is created when promoting the first DC. The forest root domain (jay.local) is the top of the entire AD hierarchy.
📁 Organisational Units (OUs) — OUs are containers that organise objects and allow different Group Policies to be applied to different sets of users and computers.
👥 Security Groups — Used to control resource access for a department. Instead of setting permissions per user, you set them on the group — all members inherit that access automatically.
🔐 Domain Authentication — When a user logs in on any domain-joined computer, the Domain Controller verifies their credentials from the AD database — not the local machine.
📋 Naming Conventions — Following a consistent naming convention for users (e.g. first name + last initial) is a sysadmin best practice to keep accounts organised and predictable.
🔑 Group Membership — Adding a user to a Security Group grants them the permissions assigned to that group — e.g. Finance group members can access the Finance shared folder.
